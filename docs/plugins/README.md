# Plugins

Every integration in Riven is a plugin. Plugins subscribe to core events and are
configured independently, so a plugin whose required settings are missing is skipped
rather than taking down startup.

The pages below are partly generated: the settings tables come straight from each
plugin's `settings_schema()` in the code, so they cannot drift. Regenerate with:

```bash
cargo run --bin gen-docs
```

## Configuring a plugin

Settings are read from environment variables named
`RIVEN_PLUGIN_SETTING__<PLUGIN>__<setting>`, where `<PLUGIN>` is the plugin name in
uppercase:

```bash
RIVEN_PLUGIN_SETTING__TMDB__apikey="your-tmdb-key"
RIVEN_PLUGIN_SETTING__STREMTHRU__realdebridapikey="your-rd-key"
```

Values saved through the settings UI are stored in the database and layered on top of
the environment, so a change made in the UI wins over the corresponding variable.

## Enabling a plugin

By default a plugin turns itself on once it has settings, so setting the variables above
is usually all that is needed. To pin the set explicitly, use an allow-list — then only
the plugins you name run:

```bash
RIVEN_SETTING__enabledPlugins=["tmdb","stremthru","torrentio","plex"]
```

`tmdb` and `tvdb` are always enabled regardless of the allow-list; metadata resolution
is not optional.

## A minimum working set

1. **Metadata** — [tmdb](tmdb.md) (always on, but it needs an API key)
2. **A content source** — [seerr](seerr.md), [mdblist](mdblist.md), [listrr](listrr.md),
   or [trakt](trakt.md)
3. **A scraper** — [torrentio](torrentio.md), [comet](comet.md),
   [aiostreams](aiostreams.md), or [newznab](newznab.md) for Usenet
4. **A downloader** — [stremthru](stremthru.md) for debrid, or [usenet](usenet.md) for NNTP
5. **A media server** — [plex](plex.md), [jellyfin](jellyfin.md), or [emby](emby.md)
