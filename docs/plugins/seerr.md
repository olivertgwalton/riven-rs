# Seerr

Content source backed by [Overseerr](https://overseerr.dev) or
[Jellyseerr](https://github.com/Fallenbagel/jellyseerr). Riven polls the request list and
pulls in anything matching the filter, so users keep requesting through the interface they
already use.

The `filter` setting takes an Overseerr request status and defaults to `approved`, meaning
requests are only picked up once someone has approved them. Set it to `all` if you want
Riven to act on pending requests too.

Requests are paged 20 at a time. `url` must be reachable from the Riven container.
