# Webhooks

Outbound event bus. Every notable state transition is POSTed as a signed JSON envelope to
each configured URL, which is what you want for wiring Riven into Home Assistant, n8n,
ntfy, or your own scripts. For human-readable download alerts, [notifications](notifications.md)
is the simpler choice.

Delivery is **at-least-once**. Transient failures are retried by the shared HTTP client,
and a delivery that still fails lands in a capped Redis dead-letter list rather than being
dropped silently. Two consequences for consumers:

- Deduplicate on the envelope `id` — the same event can arrive twice.
- Do not rely on ordering. Hook jobs run concurrently, so a `download.success` can arrive
  before the `scrape.success` that preceded it.

Payloads are signed so receivers can verify the sender; `urls` is comma-separated.

### Events

Subscribes to:

- `riven.media-item.download.success`
- `riven.media-item.scrape.success`
- `riven.media-item.index.success`
- `riven.media-item.download.error`
- `riven.media-item.scrape.error`
- `riven.media-item.scrape.error.no-new-streams`
- `riven.item-request.created`
- `riven.item-request.updated`

### Settings

Set via environment variables prefixed with `RIVEN_PLUGIN_SETTING__WEBHOOKS__`, or in the settings UI.

| Setting | Environment variable | Type | Required | Default | Description |
| --- | --- | --- | --- | --- | --- |
| `urls` | `RIVEN_PLUGIN_SETTING__WEBHOOKS__urls` | text (multiline) | Yes | — | Comma-separated URLs. Each receives a POST request when an event fires. |
| `events` | `RIVEN_PLUGIN_SETTING__WEBHOOKS__events` | string list (`riven.media-item.download.success`, `riven.media-item.scrape.success`, `riven.media-item.index.success`, `riven.media-item.download.error`, `riven.media-item.scrape.error`, `riven.media-item.scrape.error.no-new-streams`, `riven.item-request.created`, `riven.item-request.updated`) | No | — | Which events to send. Leave empty to send all events. |
| `secret` | `RIVEN_PLUGIN_SETTING__WEBHOOKS__secret` | secret | No | — | Optional secret key. When set, each request includes a signature header so you can verify it came from Riven. |
| `enrich` | `RIVEN_PLUGIN_SETTING__WEBHOOKS__enrich` | boolean | No | — | Include full item details (title, artwork, rating) in the payload. |
