# Notifications

Sends a notification when a download succeeds. Discord webhook URLs get a rich embed;
anything else receives generic JSON, so ntfy, Gotify, and homegrown endpoints all work.

`urls` is comma-separated, so one Riven instance can notify several places at once.
Turning on `detailed` adds the overview text, rating, and external links to Discord
embeds — the overview specifically comes from TMDB, so that mode also needs
`tmdb_api_key` (a TMDB *read access token*, not the API key used by the
[tmdb](tmdb.md) plugin).

For automation rather than human-readable alerts, use [webhooks](webhooks.md) instead —
it emits the raw event stream rather than one rendered message.

### Settings

Set via environment variables prefixed with `RIVEN_PLUGIN_SETTING__NOTIFICATIONS__`, or in the settings UI.

| Setting | Environment variable | Type | Required | Default | Description |
| --- | --- | --- | --- | --- | --- |
| `urls` | `RIVEN_PLUGIN_SETTING__NOTIFICATIONS__urls` | text (multiline) | Yes | — | Comma-separated webhook URLs. Supports Discord and generic JSON endpoints. |
| `detailed` | `RIVEN_PLUGIN_SETTING__NOTIFICATIONS__detailed` | boolean | No | — | Show rich Discord embeds with overview, rating, and external links. |
| `tmdb_api_key` | `RIVEN_PLUGIN_SETTING__NOTIFICATIONS__tmdb_api_key` | secret | No | — | Optional. Required for overview text in detailed Discord embeds. |
