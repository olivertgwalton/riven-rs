# StremThru

The debrid layer. [StremThru](https://github.com/MunifTanjim/stremthru) fronts every
supported debrid service behind one API, and this plugin uses it for the whole download
half of the pipeline: checking whether a torrent is already cached, sending it to the
service, and resolving the final stream link the VFS reads from.

Configure at least one provider API key. Riven treats every key you set as an available
store and scores them over a rolling week, preferring whichever has been serving cached
content most reliably.

Two settings are worth understanding before you change them:

- **`checkdebridcache`** (on by default) restricts downloads to torrents already cached by
  the service. Turning it off makes Riven request uncached torrents too, which means
  waiting on the service to fetch them — useful for obscure content, slow for everything else.
- **`scrapenabled`** controls whether StremThru is also used as a *scraper*. You can leave
  it off and rely on [Torrentio](torrentio.md), [Comet](comet.md), or
  [AIOStreams](aiostreams.md) for search while still using StremThru to download.

Setting `newznabenabled` additionally routes Usenet searches through StremThru, which
requires `stremthruauth`. For NNTP downloading without StremThru, see [usenet](usenet.md).
